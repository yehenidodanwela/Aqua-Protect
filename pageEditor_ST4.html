Profile

Technical Description

The div tag was heavily used to structure content into clear sections such as the profile card, contact form, and skills list. Unordered lists (ul with li) were used to represent skills and interests, ensuring visual clarity. A table was implemented to display a weekly availability schedule in a grid-like structure for easy understanding. The form tag allowed users to input and submit their information, showcasing interactive form functionality.The layout relied on display: flex; to organize sections side by side, making it both structured and responsive. The profile image used border-radius: 50% to appear circular, adding a modern aesthetic. Hover effects (:hover) on interactive elements created a responsive, dynamic feel, enhancing user engagement.

Accessibility

ll images across pages included descriptive alt attributes, improving accessibility for screen reader users. Pages used clear semantic tags (header, nav, main, section) and div structures to enhance compatibility with assistive technologies.

Sitemap

Technical Description

An svg element formed the core of this page to visually display the site structure. Clickable nodes were created using rect tags, and connections between these nodes were illustrated using line elements. This use of scalable vector graphics provided a clear, responsive, and visually engaging sitemap that reflected the hierarchy and navigation of the site.

Accessibility

The SVG sitemap was designed with both visual clarity and keyboard accessibility in mind, supporting inclusive interaction.

Challenges and Lessons Learned

One major challenge was making the SVG-based sitemap responsive. Initially, it failed to scale on smaller screens, but by applying viewBox and preserveAspectRatio, the issue was resolved, greatly enhancing usability. Another challenge was maintaining a consistent visual style across pages contributed by different team members. Conflicting CSS styles led to inconsistent typography and layout, which were resolved through teamwork and the use of a shared global stylesheet.This project reinforced the value of responsive design, clean code structure, and communication within a collaborative team setting. Testing frequently, discussing solutions early, and using version control effectively allowed the team to build a cohesive and accessible site.

Compliance

The web pages follow JISC/Janet web publishing guidelines by ensuring accessibility, proper copyright attribution, and appropriate image use through free or credited sources. Each page was tested for HTML and CSS validation. The design respects privacy and content publication policies by linking to external privacy, cookie, and license agreements through the footer. Consistency in layout, navigation, and branding across pages meets internal web usability standards.
