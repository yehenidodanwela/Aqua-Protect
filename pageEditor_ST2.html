Home Page

Technical Description

The homepage uses a combination of semantic HTML elements (header, main, section, footer) and flexbox-based CSS layouts for structure and alignment. A full-screen hero section (.homepage) includes a background GIF, centrally aligned headings, and a call-to-action button using flex, align-items, and justify-content. Reusable buttons are styled using classes like .explore-btn with hover effects and smooth transitions. The content cards are placed in a horizontal flex container (.contents) and respond to smaller screens with media queries. Styling is modular, with a Global_CSS/Common.css file to ensure consistency.

Accessibility

Semantic tags (nav, header, main, footer) improve screen reader navigation. All images include descriptive alt attributes. Clear text hierarchy, meaningful button labels, and high-contrast color choices enhance readability. Responsive typography and text-shadow enhance visibility against the animated GIF background.

Table Page

Technical Description

This page utilizes a semantically structured table with accessible elements like aria-label, role="columnheader", and tooltips for icons to provide clarity. The layout is centered using a .table-container flexbox wrapper. CSS enhances visuals with alternating row colors, hover effects, icons in column headers, and responsive behavior. Each organization is displayed with logical grouping using rowspan and color-coding. Verified organizations are visually marked with icons and styled with hoverable links.

Accessibility

This page uses aria-label attributes on navigation and table headers to guide assistive technologies. Icon headers include tooltips for further clarity, and verified checkmarks are visually and textually represented. Color contrast and hover states are keyboard-friendly and meet WCAG standards.

Content Page

Technical Description

The content page is split into thematic sections using descriptive section tags and internal navigation via a toggleable sidebar. Hero sections are styled with full-width background images, and key data is shown in structured cards (.card) and grids. Responsive layouts adapt using flex-wrap and media queries. The before-after section creatively uses paired images to highlight success stories, while list elements like .policieslist are styled with custom borders and background for better readability.

Accessibility

Each card and interactive element includes descriptive alt text. The toggleable sidebar navigation enables quick internal page access. Layouts use semantic structure, and paired images in the before-after section include context for screen readers. Text contrasts, structured lists, and meaningful headings support accessibility.

Challenges and Lessons Learned

Challenges included managing responsive design across three different pages, ensuring consistent layout without breaking functionality on smaller devices. One key learning was how to apply flexbox and media queries effectively. Accessibility also presented a learning curve, especially with implementing aria-labels and tooltip guidance. Adapting layout for both aesthetic appeal and semantic correctness helped in writing cleaner, more maintainable code.

Compliance

The web pages follow JISC/Janet web publishing guidelines by ensuring accessibility, proper copyright attribution, and appropriate image use through free or credited sources. Each page was tested for HTML and CSS validation. The design respects privacy and content publication policies by linking to external privacy, cookie, and license agreements through the footer. Consistency in layout, navigation, and branding across pages meets internal web usability standards.
