Splash Page

Technical Description

The Splash Page uses a meta tag to automatically redirect users to the homepage after a delay of four seconds. The page structure is managed using a splash-container class that keeps the content centered on the screen. A loader-container wraps the visual loading spinner, while the loader class handles the animated spinner effect through CSS. The animation is created using CSS keyframes to rotate the loader 360 degrees continuously. The body of the page uses a flex layout with centered alignment and a height of 100vh to fill the entire screen. Additional styling includes background settings such as no-repeat and cover to ensure the splash screen appears clean and full-screen. The splash screen container uses border-radius for rounded edges and flexbox for vertical alignment of its elements. The button on the splash screen is styled with padding, font adjustments, and transitions for hover and click effects. When hovered, the background turns blue, and when clicked, the button shrinks slightly, offering a responsive feel.

Accessibility

The Splash Page is designed with accessibility in mind by using high-contrast colors and large font sizes for improved readability. Buttons are focusable and include hover and active states to support users navigating with keyboards. The HTML structure supports screen readers with descriptive content and semantic layout. The spinner animation rotates slowly to reduce motion sensitivity, and the layout is responsive across different screen sizes. In addition to the auto-redirect, a manual button is provided to give users more control and navigation flexibility.

Table Page

Technical Description

The Volunteer Page starts with a navigation bar created using a navigation tag. Inside it, the site logo is wrapped in an anchor link pointing to the homepage. A spacer class is used to separate the logo from the menu items, which are listed inside an unordered list. A filter section follows, using a label and select dropdown for users to sort volunteer activities. The activities themselves are displayed inside an activities-container, with each one enclosed in an activity-box. Buttons inside these boxes allow users to view projects or apply directly. The review section includes a review-summary class to display an overall rating, a heading to show the review score, and a paragraph with a see more link. Finally, a feedback form at the bottom of the page includes a textarea for users to share their thoughts. CSS is used throughout to structure the layout, using flex display, flex-wrap for responsiveness, and gap to space the boxes. The review section uses a light background and rounded corners to stand out. Buttons are styled with pointer cursors and slight border radius for accessibility and interaction feedback.

Accessibility

The Volunteer Page uses semantic HTML to structure content clearly, which benefits screen readers and assistive technologies. All images are given descriptive alt attributes, and form elements like the filter dropdown are associated with labels. The layout allows full keyboard navigation, with all buttons and links accessible via the Tab key. The design ensures high color contrast between text and background, and all text is scalable using relative units. Skip links are implemented to help users quickly jump to key sections, improving navigation for users with mobility or visual impairments.

Content Page

Technical Description

Content Page 2 begins with a header that includes metadata, links to external stylesheets, and a Google font. The sidebar menu is toggled using a hidden checkbox input. When activated, the sidebar slides into view and provides navigation links to sections within the page. Each piece of content is displayed inside a card element, arranged in a threat-grid container. These cards present different environmental threats, along with related actions and images. The page uses buttons to encourage user engagement, such as applying or learning more. Background images are applied to hero sections using CSS properties like no-repeat, center, and cover. The sidebar uses CSS transitions for smooth sliding, while the main content is styled using flex and grid layouts to ensure responsiveness. Cards include hover effects and shadows for interactivity. A floating back-to-top button sits in the bottom-right corner of the screen, making it easy for users to navigate back to the top.

Accessibility

Content Page 2 ensures that all images include descriptive alt attributes, making them accessible to screen readers. Text is displayed with adequate contrast against background images to support users with visual impairments. Fonts are sized using scalable units like rem and percentages, allowing users to zoom or resize text as needed. Hyperlinks use clear and descriptive text to guide users, avoiding vague phrases like “click here.” The layout is responsive and adapts to mobile devices through flexible grids and media queries, ensuring accessibility across all screen sizes and input types.

Challenges and Lessons Learned

One of the main challenges faced was organizing a large amount of content in a way that is easy to navigate. This was solved by introducing a sticky sidebar and anchor links, allowing users to jump to specific sections. Cards and grids were used to break content into smaller, more digestible pieces. Maintaining consistent design across multiple pages was also challenging, especially when different styles were applied. A common CSS file was introduced to unify the design. Another issue was ensuring images were responsive across screen sizes, which was solved by using flexbox and setting maximum width properties. Key lessons learned include the importance of accessibility, such as using aria labels and alt attributes, and optimizing performance through image compression and lazy loading. Ensuring responsive design and consistent code structure helped improve the user experience. These improvements resulted in a more user-friendly, accessible, and efficient website.

Compliance

The Splash, Volunteer, and Content Page 2 all meet Janet compliance guidelines. Accessibility is ensured through the use of semantic HTML, proper contrast, keyboard navigability, and screen reader compatibility. Data protection is achieved by obtaining user consent before collecting any personal information and securely storing only necessary data. The site follows security standards by implementing HTTPS, input validation, CAPTCHA, and content security policies to prevent threats like cross-site scripting. Performance is optimized by compressing assets, minifying code, and designing for mobile responsiveness. Together, these measures ensure that the site is safe, fast, accessible, and legally compliant.
